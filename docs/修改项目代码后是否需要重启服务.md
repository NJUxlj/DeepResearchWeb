## 修改项目代码后是否需要重启服务？

### 后端 (Backend)

| 文件/目录 | 更改类型 | 是否需要重启 | 说明 |
|-----------|----------|-------------|------|
| `app/config.py` | Python 代码 | ✅ 重启后端 | 配置类代码变更 |
| `app/services/*.py` | Python 代码 | ✅ 重启后端 | 业务逻辑代码变更 |
| `app/api/*.py` | Python 代码 | ✅ 重启后端 | API 路由代码变更 |
| `app/models/*.py` | Python 代码 | ✅ 重启后端 | 数据模型变更 |
| `requirements.txt` | 依赖包 | ✅ 重新构建镜像 | 新增 Python 包 |
| `.env` | 环境变量 | ✅ 重启后端 | 配置文件变更 |

### 前端 (Frontend)

| 文件/目录 | 更改类型 | 是否需要重新构建 | 说明 |
|-----------|----------|-----------------|------|
| `src/pages/*.tsx` | React 页面 | ✅ 重新构建镜像 | 新增或修改页面组件 |
| `src/components/*.tsx` | React 组件 | ✅ 重新构建镜像 | 新增或修改 UI 组件 |
| `src/api/*.ts` | API 客户端 | ✅ 重新构建镜像 | 新增或修改 API 调用 |
| `src/stores/*.ts` | 状态管理 | ✅ 重新构建镜像 | Zustand 等状态管理 |
| `src/hooks/*.ts` | React Hooks | ✅ 重新构建镜像 | 新增或修改 hooks |
| `src/types/*.ts` | TypeScript 类型 | ✅ 重新构建镜像 | 类型定义变更 |
| `.env` | 环境变量 | ✅ 重新构建镜像 | 前端环境变量 |
| `package.json` | 依赖包 | ✅ 重新构建镜像 | 新增 npm 包 |
| `tailwind.config.ts` | 配置 | ✅ 重新构建镜像 | Tailwind 配置 |
| `vite.config.ts` | 配置 | ✅ 重新构建镜像 | Vite 配置 |

### 文档

| 文件/目录 | 更改类型 | 是否需要重启 | 说明 |
|-----------|----------|-------------|------|
| `README.md` | 文档 | ❌ 不需要 | 文档变更 |
| `docs/*.md` | 文档 | ❌ 不需要 | 文档变更 |

## 重启命令

### 后端服务

```bash
# 仅重启后端
docker-compose restart backend

# 重新构建并启动后端（修改了 requirements.txt 或 Python 代码）
docker-compose up -d --build backend

# 仅强制重新构建后端
docker-compose build --no-cache backend
```

### 前端服务

```bash
# 仅重启前端
docker-compose restart frontend

# 重新构建并启动前端（修改了前端代码）
docker-compose up -d --build frontend
```

### 同时重启后端和前端

```bash
# 重启所有应用服务
docker-compose restart backend frontend

# 重新构建所有应用服务
docker-compose up -d --build backend frontend
```

## 常见场景

### 场景 1: 修改了后端 .env 变量

```bash
# 例如修改了 USE_MEMOS=true
# 只需重启后端
docker-compose restart backend
```

### 场景 2: 修改了前端页面代码

```bash
# 例如添加了新的配置页面
# 需要重新构建前端
docker-compose up -d --build frontend
```

### 场景 3: 修改了 requirements.txt

```bash
# 例如添加了 MemoryOS 包
# 需要重新构建后端
docker-compose up -d --build backend
```

### 场景 4: 修改了后端 Python 代码

```bash
# 例如修改了 memory_service.py
# 只需重启后端
docker-compose restart backend
```

---

## 注意事项

1. **修改 `.env` 文件后必须重启服务**，否则容器内的环境变量不会更新
2. **前端代码变更需要重新构建镜像**，因为前端是打包后部署的
3. **后端 Python 代码变更只需重启**，不需要重新构建（除非依赖变化）
4. 使用 `--build` 参数会强制重新构建镜像，即使没有代码变化
