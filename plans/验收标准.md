# DeepResearchWeb 验收标准

本文档定义了 DeepResearchWeb 项目的验收标准。所有代码构建完成后，必须按照以下步骤逐一验证，确保系统正常运行。

---

## 验收流程概览

| 步骤 | 验证内容 | 预期结果 |
|------|----------|----------|
| 1 | Docker 容器状态 | 所有容器 running |
| 2 | 单元测试 | 无异常，全部通过 |
| 3 | API 端点测试 | 无异常，全部通过 |
| 4 | 日志检查 | 无异常错误 |

### 预期结果

- 无 `ERROR`, `EXCEPTION`, `FAILED`, `FATAL` 关键词

1. 定位异常来源（文件 + 行号）

## 完整验收执行脚本

为简化流程，可使用以下一键执行脚本：

```bash
#!/bin/bash
# acceptance_test.sh - 一键验收测试

set -e

PROJECT_DIR="/Users/xiniuyiliao/Desktop/code/DeepResearchWeb"
cd "$PROJECT_DIR"

echo "=========================================="
echo "DeepResearchWeb 验收测试"
echo "=========================================="

# Step 1: Docker 容器检查
echo ""
echo "[Step 1/4] 检查 Docker 容器状态..."
docker ps --format "table {{.Names}}\t{{.Status}}" | grep -E "running|mysql|neo4j|milvus|qdrant|rabbitmq|redis|backend|frontend"

# Step 2: 单元测试
echo ""
echo "[Step 2/4] 运行单元测试..."
cd "$PROJECT_DIR/backend"
pytest -v || { echo "单元测试失败!"; exit 1; }

# Step 3: API 测试
echo ""
echo "[Step 3/4] 运行 API 测试..."
cd "$PROJECT_DIR"
bash scripts/test_all_apis.sh || { echo "API 测试失败!"; exit 1; }

# Step 4: 日志检查
echo ""
echo "[Step 4/4] 检查日志..."
cd "$PROJECT_DIR/logs"
ERRORS=$(grep -ri "error\|exception\|failed\|fatal" *.log 2>/dev/null || true)
if [ -n "$ERRORS" ]; then
    echo "发现异常日志:"
    echo "$ERRORS"
    exit 1
fi

echo ""
echo "=========================================="
echo "验收测试全部通过!"
echo "=========================================="
```

## 循环迭代流程

当代码修改后，必须重新执行完整验收流程：

```
代码修改 → Docker 重新构建 → 验收测试 → 如有异常 → 修复代码 → 重新构建 → 再次验收
```

### 迭代命令

```bash
# 1. 停止服务
docker-compose down

# 2. 重新构建
docker-compose build --no-cache

# 3. 启动服务
docker-compose up -d

# 4. 等待服务就绪
sleep 30

# 5. 重新运行验收测试
bash /Users/xiniuyiliao/Desktop/code/DeepResearchWeb/scripts/test_all_apis.sh

# 6. 检查日志
tail -n 200 /Users/xiniuyiliao/Desktop/code/DeepResearchWeb/logs/*.log
```
